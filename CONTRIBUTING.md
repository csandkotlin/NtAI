
# Contributing to NtAI

First off, thank you for considering contributing to NtAI! 🎉

NtAI is a beginner-friendly project, and all contributions are welcome — from typo fixes to new features.

## How Can I Contribute?

### Report Bugs

If you find a bug, please [open an issue](https://github.com/csandkotlin/NtAI/issues) and include:
- A clear title and description
- Steps to reproduce the bug
- Expected vs actual behavior
- Screenshots (if applicable)

### Suggest Enhancements

Have an idea for a new pattern, better responses, or improved code structure? Open an issue first to discuss it.

### Fix Code or Docs

1. Fork the repository
2. Create a new branch: `git checkout -b feat/your-feature-name` or `fix/bug-name`
3. Make your changes
4. Test your changes locally: `dotnet build` then `dotnet run`
5. Commit using Conventional Commits format (see below)
6. Push to your fork and open a Pull Request

## Commit Convention

This project uses [Conventional Commits](https://www.conventionalcommits.org/):

```

<type>: <subject>

```

**Types:**
- `feat:` new feature or new pattern
- `fix:` bug fix
- `docs:` documentation only changes
- `style:` formatting, missing semicolons, etc (no code change)
- `refactor:` code change that neither fixes nor adds a feature
- `test:` adding missing tests
- `chore:` maintenance tasks

**Examples:**
- `feat: add response for "thank you"`
- `fix: handle empty input correctly`
- `docs: update README with more examples`

## Pull Request Process

1. Ensure your code builds without errors
2. Update the README or docs if needed
3. Link your PR to any related issues
4. Wait for review — I'll respond within a few days

## Code Style

- Use consistent indentation (spaces or tabs, just keep it the same)
- Add comments for complex logic
- Keep methods focused and small

## Questions?

Open an issue or reach out. Thanks for helping make NtAI better! 🚀
